## 2. 사용자 스토리

1. **As a** 화주 의사결정자, **I want** 운송 요청이 OPTIC에서 어떤 순서로 처리되는지 보기를, **so that** 우리 회사 요청 방식에 적용 가능한지 판단할 수 있다.
2. **As a** 주선사 운영자, **I want** AI 오더 등록, 배차, 화물맨 연동, 정산, 세금계산서 흐름을 한 번에 이해하기를, **so that** 기존 수작업 흐름과 비교할 수 있다.
3. **As a** 도입 검토자, **I want** 화주와 주선사별로 다른 요청 양식과 정산 기준을 맞출 수 있음을 확인하기를, **so that** 단순 SaaS 화면이 아니라 맞춤형 운송 운영 서비스로 인식할 수 있다.
4. **As a** 이후 F4 구현자, **I want** 단계별 구조와 상태 후보가 먼저 확정되기를, **so that** 애니메이션과 mock 상태를 덧붙일 때 재작업을 줄일 수 있다.

## 3. 러프 요구사항

### 3.1 섹션 구조

- **R1**: 신규 섹션은 `workflow-manual` 또는 동등한 명칭으로 분리한다.
- **R2**: 섹션은 제품 라인업 카드가 아니라 업무 단계형 매뉴얼로 구성한다.
- **R3**: 단계 순서는 `AI 오더 등록 → 상하차지 관리 → 배차/운송 상태 → 화물맨 연동 → 정산 자동화 → 세금계산서 관리`를 기본값으로 둔다.
- **R4**: 각 단계는 `무엇을 입력하는가`, `무엇이 이어지는가`, `무엇을 회사별로 맞출 수 있는가`를 짧게 보여준다.
- **R5**: 섹션은 F2의 제품 라인업과 중복되지 않도록 `업무 순서`와 `운영 흐름`에 집중한다.

### 3.2 카피 기준

- **R6**: `OPTIC`의 핵심 의미는 약어 풀이가 아니라 "화주와 주선사마다 다른 운송 운영을 하나의 흐름으로 조율한다"는 가치 문장으로 표현한다.
- **R7**: 화주별 요청 양식, 주선사별 배차 방식, 담당자/승인 흐름, 정산 기준, 외부 채널 연동을 맞출 수 있다는 메시지를 포함한다.
- **R8**: `화물맨 연동`은 배차 단계에서 한 번 입력한 운송 정보를 외부 채널로 이어 보내는 기능으로 설명한다.
- **R9**: 정산 파트는 `정산 자동화`, `세금계산서 관리`로 표현한다.
- **R10**: 설정 UI, tenant admin, 실제 API 연동이 이미 구현된 것처럼 오해될 수 있는 문구는 피한다.

### 3.3 UX/Responsive 기준

- **R11**: desktop에서는 단계형 timeline 또는 split rail 구조를 우선 검토한다.
- **R12**: mobile에서는 카드가 세로로 쌓이고, 단계 번호/라벨/본문이 겹치지 않아야 한다.
- **R13**: 375px 폭에서 긴 한글 단어와 영문 보조 라벨이 버튼이나 카드 밖으로 넘치지 않아야 한다.
- **R14**: F4 이전 단계이므로 motion은 최소화하고, 상태 표현은 static 또는 아주 제한적인 hover/focus 수준으로 둔다.

### 3.4 Out-of-scope

- 실제 화물맨 API 연동
- 실제 정산/세금계산서 API 연동
- 화주/주선사별 설정 저장 구조
- tenant admin 또는 고객별 설정 UI
- F4의 스크롤 애니메이션, stagger, 상태 mock 고도화
- F5의 favicon, Open Graph, 브랜드 자산 확정

## 4. 업무 흐름 초안

| 단계 | 표시 제목 | 핵심 메시지 | 커스텀 포인트 |
|---:|---|---|---|
| 1 | AI 오더 등록 | 여러 형식의 운송 요청을 AI가 오더 정보로 정리한다 | 화주별 요청 양식, 필수 입력값 |
| 2 | 상하차지 관리 | 자주 쓰는 장소, 담당자, 메모를 반복 입력 없이 재사용한다 | 장소명, 담당자, 현장 메모 |
| 3 | 배차/운송 상태 | 배차 진행과 운송 상태를 한 흐름에서 확인한다 | 주선사별 상태명, 승인 흐름 |
| 4 | 화물맨 연동 | 배차 단계에서 운송 정보를 외부 채널로 이어 보낸다 | 연동 시점, 전송 필드, 확인 방식 |
| 5 | 정산 자동화 | 운송 완료 후 매출 정산 기준을 자동으로 묶어 관리한다 | 화주별 청구 기준, 주선사별 정산 기준 |
| 6 | 세금계산서 관리 | 정산 이후 증빙 상태까지 놓치지 않게 이어서 확인한다 | 발행 상태, 담당자 확인, 후속 알림 |

## 6. Acceptance Criteria

- 업무 매뉴얼형 섹션이 랜딩에 추가되고, 6단계 흐름이 순서대로 노출된다.
- 각 단계는 입력, 다음 흐름, 커스텀 포인트 중 최소 2개 이상을 고객이 이해할 수 있게 설명한다.
- `화물맨 연동`은 배차 단계로 배치된다.
- `정산 자동화`, `세금계산서 관리` 문구가 정산 단계에서 명확히 노출된다.
- 화주/주선사별 커스텀 가능성이 섹션 headline 또는 보조 문구와 각 단계 copy에 반영된다.
- 기존 제품 라인업과 중복된 제품 카드 구조를 만들지 않는다.
- 375px mobile, tablet, desktop에서 텍스트 겹침이 없다.
- 신규 섹션이 F4 애니메이션 작업을 이어받을 수 있도록 단계 데이터와 컴포넌트 구조가 분리된다.
