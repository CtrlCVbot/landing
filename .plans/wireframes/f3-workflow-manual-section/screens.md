## 2. SCR-F3-001 — Workflow Manual Section Desktop

### 목적

방문자가 OPTIC의 주 서비스를 제품명보다 실제 업무 흐름으로 이해하게 한다. desktop에서는 왼쪽에 가치 메시지와 CTA, 오른쪽에 6단계 timeline을 배치해 `AI 오더 등록 → 세금계산서 관리`까지의 흐름을 한 화면에서 훑을 수 있게 한다.

### Desktop ASCII

│  요청 양식, 배차 방식, 정산 기준, 외부 채널 연동까지 회사별 방식에 맞춥니다. │

│  ┌──────────────────────────────┐  ┌──────────────────────────────────────┐ │

### Desktop State Rules

| 상태 | 표현 | 동작 |
|---|---|---|
| default | 모든 단계가 같은 밀도로 보임 | scroll 위치와 관계없이 내용 이해 가능 |
| hover | 단계 row의 border 또는 background만 강조 | 높이 변경 없음 |
| focus | CTA와 interactive element에 visible focus ring | keyboard tab 순서 보장 |
| reduced motion | transition 없이 static layout 유지 | F4 애니메이션이 없어도 정보 전달 가능 |

### Mobile Guard

| 항목 | 기준 |
|---|---|
| 최소 폭 | 375px 기준 텍스트 겹침 없음 |
| 카드 높이 | 내용에 따라 늘어나되 hover/focus로 높이 변경 없음 |
| 단계 번호 | 제목 위 또는 같은 줄 왼쪽에 두되 제목을 밀어내지 않음 |
| CTA | 섹션 하단 1개 우선, 두 개 이상이면 vertical stack |
| 긴 문구 | `세금계산서 관리`, `화주/주선사별`은 강제 한 줄 고정 금지 |

## 4. SCR-F3-003 — Landing Scroll Integration

### 목적

F3는 F2 제품 라인업을 반복하지 않고, 제품 설명 다음에 실제 업무 흐름을 보여주는 섹션으로 배치한다. 배치 후보는 `Products` 이후, `Features` 또는 `Integrations` 이전이다. 기본 추천은 **Products 직후**다.

### Scroll Flow ASCII

```text
┌──────────────────────────────┐
│ Hero                         │  브랜드/CTA/서비스 진입
├──────────────────────────────┤
│ Problems / Features          │  수작업, 누락, 핵심 기능 가치
├──────────────────────────────┤
│ Products                     │  주선사용 / 화주용 제품 구분
├──────────────────────────────┤
│ Workflow Manual (F3)         │  실제 업무가 이어지는 6단계 흐름
├──────────────────────────────┤
│ Integrations                 │  화물맨 등 연동 설명 보조
├──────────────────────────────┤
│ Final CTA / Footer           │  문의 또는 서비스 이동
└──────────────────────────────┘
```

### 배치 결정

| 후보 | 판정 | 이유 |
|---|---|---|
| Products 직후 | 추천 | 제품이 누구를 위한 것인지 본 뒤, 실제 업무 순서를 자연스럽게 확인할 수 있다. |
| Hero 직후 | 보류 | 핵심 가치 전달은 빠르지만 제품 라인업을 보기 전에 단계 정보가 과하게 느껴질 수 있다. |
| Integrations 직후 | 비추천 | 화물맨/연동 설명을 본 뒤에 업무 흐름을 보면 흐름의 중심성이 약해진다. |

## 5. 반응형 기준

| Viewport | 레이아웃 | 기준 |
|---|---|---|
| Desktop 1280px+ | intro/CTA 1열 + timeline 1열 split | 한 화면에서 6단계 흐름과 커스텀 포인트를 비교 |
| Tablet 768px-1279px | intro 상단 + timeline 2열 또는 single column | 가로 폭이 좁으면 timeline을 1열로 전환 |
| Mobile 375px-767px | intro + 단계 카드 단일 열 | 단계 번호/제목/본문/맞춤 포인트 겹침 없음 |
