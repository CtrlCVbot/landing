## 1. 문제 요약

F1~F4로 OPTIC 랜딩의 브랜드명, 제품 카피, 업무 흐름 섹션, 상태 표현은 정리되었다. 하지만 공개 전 마무리 기준인 로고 파일, favicon, Open Graph 이미지, 검색/공유 메타데이터, 링크 접근성 라벨, 검증 기록이 아직 한 묶음으로 고정되지 않았다.

## 2. 목표

- header/footer에서 쓰는 OPTIC 로고 표현과 정적 브랜드 자산 기준을 맞춘다.
- favicon과 Open Graph 이미지 경로를 `metadata`에 연결한다.
- 화주/주선사별 맞춤 운송 운영 메시지가 title/description에 반영되게 한다.
- 외부 링크의 `target`/`rel`/접근성 라벨을 점검한다.
- typecheck, test, lint, build, 문구 스캔, responsive preview 결과를 남긴다.

## 3. 예상 범위

| 영역 | 예상 작업 |
|---|---|
| Assets | `public/brand/**`, `public/favicon.svg` |
| Metadata | `src/app/layout.tsx`, `src/lib/constants.ts` |
| UI | `header.tsx`, `footer.tsx`, `OpticLogo` |
| Tests | constants/layout/header/footer tests |
| QA | 문구 스캔, desktop/mobile preview |

## 4. Acceptance Criteria

- `BRAND_ASSETS`와 `SITE_METADATA` 기준이 코드에서 재사용된다.
- `favicon.svg`, `optic-logo.svg`, `optic-mark.svg`, `optic-og.svg`가 존재한다.
- `metadata`가 favicon과 Open Graph 이미지를 참조한다.
- header/footer의 로고는 접근 가능한 이름을 가진다.
- 외부 링크는 `target="_blank"`와 `rel="noopener noreferrer"`를 가진다.
- 검증 결과가 active feature package에 남는다.

## 5. 승인 기록

사용자 요청에 따라 P1~P7을 이어서 진행한다. F5는 신규 화면이 아니라 release readiness 성격이므로 Lite lane으로 승인 처리한다.
