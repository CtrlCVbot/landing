## 1. 설명

landing page의 `Hero` 섹션 배경에 CodePen의 `Interactive Liquid Gradient using Three.js` 스타일을 참고한 animated liquid gradient 효과를 도입하는 아이디어다.

참고 링크:
- https://codepen.io/cameronknight/pen/ogxWmBP

이번 아이디어의 범위는 hero 배경 레이어에만 한정한다. CodePen의 heading, color scheme buttons, color adjuster panel, footer, custom cursor, 별도 control UI는 가져오지 않는다.

## 2. 현재 프로젝트 맥락

현재 hero 진입점:
- `src/components/sections/hero.tsx`

현재 배경 요소:
- `src/components/shared/gradient-blob.tsx`

현재 `Hero`는 정적인 `GradientBlob` 2개를 배경 장식으로 사용하고, 같은 섹션 안에 `DashboardPreview`를 포함한다. 새 배경은 headline, CTA button, `DashboardPreview` 뒤에 깔리는 보조 레이어여야 하며, 텍스트 가독성이나 클릭 가능 영역을 방해하면 안 된다.

## 3. 기대 효과

| 기대 효과 | 이유 |
|---|---|
| 첫 화면 인상 강화 | 전체 레이아웃을 바꾸지 않고도 hero가 더 고급스럽고 기억에 남는 인상을 줄 수 있다. |
| AI/product 느낌 강화 | 은은한 움직임이 현재 dashboard preview와 맞물려 더 동적인 제품 경험처럼 보일 수 있다. |
| 재사용 가능한 visual system 후보 | 배경 레이어로 잘 분리하면 이후 주요 섹션의 accent visual로 확장할 수 있다. |

## 4. 현재 프로젝트 적용 방향

추천 방향은 단계적 보강이다.

1. 현재 hero copy, CTA, `DashboardPreview` 배치는 유지한다.
2. `GradientBlob`을 즉시 제거하지 말고 fallback 또는 reduced-motion 상태의 대체 요소로 둔다.
3. 먼저 CSS/canvas 기반의 가벼운 후보를 검토한 뒤, 필요할 때만 `Three.js` / WebGL shader 방향으로 확장한다.
4. 새 배경은 `pointer-events: none` 성격의 장식 레이어로 설계한다.

이렇게 하면 visual 개선이 hero 전체 재설계로 커지는 것을 막을 수 있다.

## 5. 이후 검토할 구현 후보

| 후보 | 적합도 | trade-off |
|---|---|---|
| CSS gradient + blur layer | MVP 후보로 가장 적합 | 가장 가볍지만 CodePen 같은 유동적인 질감은 제한적이다. |
| Canvas 2D animation | 중간 경로 | CSS보다 제어력이 높고 WebGL보다 부담이 낮다. |
| SVG/filter 기반 배경 | fallback 후보 | 고급 질감을 낼 수 있지만 filter가 무거워질 수 있다. |
| `Three.js` / WebGL shader | reference와 가장 유사 | 효과는 강하지만 bundle size, 성능, 유지보수 부담이 크다. |

## 6. UX 고려사항

- headline, subtitle, CTA, `DashboardPreview` 가독성을 해치지 않아야 한다.
- 배경이 클릭이나 hover 이벤트를 가로채면 안 된다.
- 움직임은 시각 데모가 아니라 business landing page에 어울리는 수준으로 절제한다.
- 모바일에서는 더 조용한 animation 또는 static fallback을 우선 검토한다.
- `prefers-reduced-motion` 사용자는 animation을 끄거나 크게 줄인다.
- 색상은 CodePen palette를 그대로 복사하지 말고 현재 landing theme에 맞게 조정한다.

## 7. 구현 전 확인해야 할 리스크

| 리스크 | 수준 | 메모 |
|---|---|---|
| 모바일 성능 저하 | high | WebGL 또는 과한 blur는 저사양 기기에서 hero 로딩과 스크롤 감각을 해칠 수 있다. |
| 전환 목적 약화 | high | 밝거나 움직이는 영역이 CTA와 headline을 가리면 landing page의 목적이 흐려진다. |
| 범위 확장 | medium | CodePen의 control UI까지 가져오면 배경 개선이 별도 tool 구현으로 커질 수 있다. |
| 외부 코드 license 불명확 | medium | CodePen은 visual reference로만 보고, 코드 재사용은 별도 확인이 필요하다. |
| bundle size 증가 | medium | `Three.js`가 새 dependency가 되면 build size와 초기 로딩 비용을 확인해야 한다. |
