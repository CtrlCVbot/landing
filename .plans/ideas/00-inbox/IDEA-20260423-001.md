## 2. 배경 (Why)

두 이슈는 별개의 UI 잔재 정리 작업이지만, 다음 이유로 단일 Feature 로 묶는다:

1. **스코프가 모두 "숨김/치환" 계열**: 신규 기능 추가가 아닌 렌더 경로 제외·라벨 교정. Lite Lane 에 적합.
2. **F2 Mock 스키마 재설계의 연쇄 정리 기반 마련**: F5 가 `ai-panel/index.tsx` 렌더 + `hit-areas.ts` · `mock-data.ts` tooltip 엔트리를 먼저 제거해두면, F2 의 `extractedFrame`/`appliedFrame` 스키마 재설계가 정리된 mock 파일 위에서 깔끔히 얹힘. `jsonViewerOpen` 필드 자체는 F5 에서 유지 → F2 재설계 시 스키마 내부로 일괄 이관 (Epic Children Features 의존성 매트릭스 F2 ↔ F5 `→` 참조).
3. **병렬 실행 가능성**: F1 (라이트 모드) 의 전역 토큰 스윕과 파일 범위가 겹치지 않아 Phase A 에 F1 과 병렬 투입 가능 (Epic F1 ↔ F5 `✓`).

## 4. 기대 효과

1. **시네마틱 데모 흐름 회복**: AI 패널 하단의 디버그성 JSON 뷰어를 화면에서 감추어 사용자가 "AI 추출 → 폼 적용" 내러티브에 집중 가능.
