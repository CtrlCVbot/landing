## 1. 문제 요약

F2에서 mock scenario와 `formData.options` 구조가 정착되었지만, 정산 정보의 추가요금은 아직 옵션 선택값과 명확히 연결된 파생값으로 고정되지 않았다. 이 상태가 유지되면 demo-safe scenario가 바뀔 때 운송 옵션 checked 상태와 정산 추가요금 표시가 서로 어긋나 보일 수 있다.

## 2. 목표

- `OPTION_FEE_MAP`을 도입해 운송 옵션 ID와 추가요금 항목을 1:1로 추적한다.
- `formData.options` 변화에 따라 `additionalFees`를 파생한다.
- `SettlementSection` 또는 `order-form/index.tsx`가 고정 mock 추가요금보다 파생 결과를 우선 사용하도록 정리한다.
- demo-safe random scenario 3개 이상에서 옵션 checked 값과 정산 추가요금 표시가 항상 일치하도록 테스트한다.

## 3. 예상 범위

| 영역 | 예상 작업 |
|---|---|
| Mock data | `src/lib/mock-data.ts`에 `OPTION_FEE_MAP` 및 파생 helper 추가 |
| Order form | `formData.options` 기반 `additionalFees` 계산 연결 |
| Settlement | 정산 정보의 추가요금 source를 파생 결과 기준으로 전환 |
| Test | option checked 상태, 추가요금 표시, 합계 정합성 테스트 추가 |
| Preview QA | scenario loop에서 옵션과 정산 추가요금 일치 여부 수동 확인 |

## 6. Acceptance Criteria

- `OPTION_FEE_MAP`의 key가 `formData.options` option id와 1:1로 추적된다.
- 옵션 unchecked 상태에서는 추가요금이 표시되지 않는다.
- 옵션 checked 상태에서는 `OPTION_FEE_MAP` 기준 추가요금이 표시된다.
- 기본 운임, 추가요금, 정산 합계가 동일한 계산 규칙을 따른다.
- `mismatch-risk`는 random preview pool에 포함되지 않는다.
