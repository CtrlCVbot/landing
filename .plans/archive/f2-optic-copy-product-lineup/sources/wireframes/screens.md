## 2. SCR-F2-001 — Products Section Desktop

### 목적

방문자가 `OPTIC Broker/Shipper`라는 영문 제품명을 몰라도 `주선사용`, `화주용` 서비스를 먼저 이해하게 한다. 기존 5개 제품을 동등하게 보여주는 구조가 아니라, 현재 구현 대상 2개와 구현 예정 3개를 분리한다.

### Desktop ASCII


### 상태 규칙

| 상태 | 표현 | 동작 |
|---|---|---|
| 구현 대상 | 선명한 카드, `구현 대상` badge, 설명/기능 목록 표시 | 클릭 CTA가 있더라도 문의/상담 목적이어야 함 |
| 구현 예정 | muted 카드, `구현 예정` badge, 짧은 설명만 표시 | 활성 제품 탭처럼 보이지 않음 |
| Hover | 카드 elevation 또는 border 강조 | 레이아웃 크기 변경 없음 |
| Focus | keyboard focus ring 표시 | 카드 내부 CTA가 생기면 tab 순서 보장 |

### Mobile Guard

| 항목 | 기준 |
|---|---|
| 최소 폭 | 375px 기준 텍스트 겹침 없음 |
| 제목 줄바꿈 | 한글 제목은 2줄까지 허용, 카드 폭을 늘리기 위해 font-size를 viewport로 스케일하지 않음 |
| 보조 라벨 | 제목 아래 작은 텍스트 또는 badge로 표시 |
| 구현 예정 | 3개 카드가 너무 길면 bullet list 또는 compact row로 축소 |

## 4. SCR-F2-003 — Copy-Only Section Preservation

F2는 products 외 섹션의 레이아웃을 크게 바꾸지 않는다. 아래 섹션은 기존 카드/리스트 구조를 유지하고 customer-facing copy만 정리한다.

```text
┌──────────────────────────────┐
│ Problems                     │  수작업 감소 / 누락 방지 중심 문구
├──────────────────────────────┤
│ Features                     │  AI 오더 등록 / 배차 단계의 화물맨 연동 / 정산 자동화 / 세금계산서 관리 중심 문구
├──────────────────────────────┤
│ Products                     │  SCR-F2-001/002 구조 적용
├──────────────────────────────┤
│ Integrations                 │  화물맨 외 provider명 일반 기능명화
└──────────────────────────────┘
```

| Viewport | 레이아웃 | 기준 |
|---|---|---|
| Desktop 1280px+ | 구현 대상 2열 + 구현 예정 3열 compact | 역할명/라벨/설명 한눈에 비교 |

| Mobile 375px-767px | 구현 대상 1열, 구현 예정 compact list | 제목/라벨/기능 목록 겹침 없음 |
