## 4. CTA 구조

CTA는 두 종류로 분리한다.

| CTA | 역할 | 추천 위치 | 추천 문구 |
| --- | --- | --- | --- |
| 실제 서비스 이동 | 사용자가 OPTIC 서비스를 바로 열어본다 | 상단 헤더, hero 보조 CTA, 마지막 CTA | `OPTIC 바로가기` |
| 도입 문의 | 상담, 계약, 도입 논의로 이어진다 | hero 주요 CTA 또는 마지막 CTA | `도입 문의하기` |
| 기술 신뢰 보조 | OPTIC의 내부 엔진 신뢰를 짧게 보조한다 | footer 또는 About | `Powered by OPTICS` |

## 5. 로고와 헤더 적용

| 항목 | 기준 |
| --- | --- |
| 헤더 로고 | 제공 이미지 기반 `OPTIC` 워드마크 사용 |
| 로고 클릭 | 랜딩 홈 또는 페이지 최상단으로 이동 |
| 실제 서비스 버튼 | 로고와 분리해 `OPTIC 바로가기`로 표시 |
| 접근성 라벨 | 로고 링크는 `OPTIC 홈`, 서비스 버튼은 `OPTIC 주선사와 화주 서비스 열기` |
| 모바일 | 워드마크가 작게 뭉개지면 원형 심볼 단독 사용 가능 |
| 품질 기준 | 구현 전 투명 배경/SVG/다크 버전 자산 준비 |

헤더에서 로고와 CTA는 역할을 분리한다.
로고는 브랜드 인지와 홈 이동을 담당하고, `OPTIC 바로가기`는 실제 서비스 진입을 담당한다.

## 6. 실제 서비스 이동 버튼

| 항목 | 기준 |
| --- | --- |
| URL | `https://mm-broker-test.vercel.app/` |
| 링크 성격 | OPTIC 주선사와 화주 서비스가 실제로 동작하는 서버로 이동 |
| 피해야 할 문구 | `서비스 테스트`, `테스트 서버`, `데모 테스트` |
| 추천 문구 | `OPTIC 바로가기` |
| 보조 후보 | `OPTIC 시작하기`, `실제 서비스 보기`, `주선사·화주 서비스 열기` |
| 접근성 라벨 | `OPTIC 주선사와 화주 서비스 열기` |
| 링크 속성 | 외부 링크이므로 `target="_blank"`와 `rel="noopener noreferrer"` 권장 |
| 도메인 유의점 | 현재 URL은 브랜드 도메인 표준 `{service}.{tenant}.{root}`와 다르므로 공개 전 확인 필요 |

`서비스 테스트`라는 문구는 내부 검증 느낌이 강하다. 랜딩 방문자에게는 “테스트 서버”라는 운영 맥락보다 “실제 서비스를 열어본다”는 행동이 더 중요하므로 `OPTIC 바로가기`를 기본 추천한다.

## 7. CTA 배치 옵션

| 옵션 | 구조 | 장점 | 리스크 | 추천 |

| 모바일 메뉴 포함 | 모바일 overlay 안에 바로가기 표시 | 모바일 사용자도 접근 가능 | 메뉴 높이 관리 필요 | high |

## 8. 현재 문구 개선 방향

| 현재 유형 | 문제 | 개선 원칙 |
| --- | --- | --- |
| 짧은 기능명 | 기능은 보이지만 업무 가치가 약하다 | “무엇을 줄이는지”를 함께 말한다 |
| 연동명 나열 | 로고/이름만으로는 효과가 약하다 | “한 번 입력 → 외부 전송/상태 확인”으로 설명한다 |
| 정산 자동화 | 회계/정산 담당자가 원하는 구체성이 부족하다 | 매출/매입, 번들, 계산서, 입금/송금 상태를 나눠 설명한다 |
| 제품 탭 | Broker/Shipper/Carrier/Billing 구분은 있으나 흐름 연결이 약하다 | 역할별 대표 업무와 랜딩 섹션을 연결한다 |
| `OPTICS` 노출 | 엔진 브랜드를 고객 화면 전면에 쓰면 이중 브랜드처럼 보일 수 있다 | footer/About의 `Powered by OPTICS` 또는 내부 기술 설명으로 제한한다 |

## 9. 제품 라인업 문구 기준

| 제품 | 고객용 설명 |
| --- | --- |
| `OPTIC Broker` | 주선사용 운송 운영 제품 |
| `OPTIC Shipper` | 화주용 운송 요청/현황 확인 제품 |
| `OPTIC Carrier` | 운송사/차주용 배차 수행 제품 |
| `OPTIC Ops` | 운영/관제용 모니터링 제품 |
| `OPTIC Billing` | 정산/세금계산서 관리 제품 |

고객용 화면에서는 위 이름을 그대로 사용한다. 내부 구현 문서에서 정산 계산 근거가 필요할 때만 `OPTICS Ledger`를 함께 언급한다.
