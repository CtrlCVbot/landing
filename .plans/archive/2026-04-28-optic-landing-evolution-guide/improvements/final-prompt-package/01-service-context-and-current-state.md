# 01. 서비스 이해와 현재 랜딩 진단

## 1. 서비스 한 줄 정의

OPTIC은 화주가 운송 오더를 요청하고, 주선사가 배차와 운송 진행을 관리하며, 운송 완료 후 매출/매입 정산과 세금계산서까지 이어지는 업무를 한 흐름으로 묶어주는 운송 운영 SaaS 제품 브랜드다.

여기서 주선사는 화주와 기사/운송사를 연결하는 운영 주체이고, 화주는 운송을 요청하는 고객사다.

브랜드 기준은 `OPTIC = Experience(고객 경험)`, `OPTICS = Engine(내부 엔진)`이다. 따라서 이 랜딩 문서에서는 고객이 보는 이름은 `OPTIC`으로 통일하고, `OPTICS`는 footer/About 또는 내부 기술 설명에서만 보조적으로 사용한다.

## 2. 핵심 사용자

| 사용자 | 하는 일 | 랜딩에서 보여줘야 할 가치 |
| --- | --- | --- |
| 주선사 운영자 | 주문 등록, AI 오더 등록, 배차 현황, 업체/기사/상하차지 관리, 매출/매입 정산 | 반복 입력을 줄이고 오더부터 정산까지 한 화면 흐름으로 관리 |
| 화주 | 주소 등록, 운송 요청, 주문 현황 확인, 정산 조회 | 요청 등록과 진행 확인을 쉽게 처리 |
| 정산/회계 담당자 | 매출 정산, 매입 정산, 세금계산서 상태 관리 | 여러 운송 건을 묶어 청구/지급하고 누락을 줄임 |
| 의사결정자 | 운영 효율, 누락 방지, 외부 연동, 비용/수익 관리 확인 | 단순 주문 관리가 아니라 운영 체계 전체를 바꾸는 서비스로 이해 |

## 4. 랜딩에서 더 설명해야 할 부분

| 부족한 설명 | 왜 중요한가 | 개선 방향 |
| --- | --- | --- |
| OPTIC이 누구를 위한 서비스인지 | 현재 기능명이 많으면 방문자가 자기 역할을 바로 찾기 어렵다 | 주선사와 화주 관점을 분리해 설명 |
| AI 오더 등록 이후 업무 흐름 | hero는 강하지만 이후 섹션이 기능 카드 중심이면 실제 운영 흐름이 끊긴다 | 스크롤을 내리며 운송 오더 이후 업무가 이어지는 구조로 설계 |
| 상하차지 관리의 가치 | 단순 주소록처럼 보이면 중요도가 낮아 보인다 | 반복 입력 감소, 담당자/메모 재사용, 주문 등록 시간 단축으로 설명 |
| 정산 번들화 | “정산 자동화”만으로는 실제 차별점이 약하다 | 여러 운송 건이 하나의 청구/지급 묶음으로 합쳐지는 장면을 보여줌 |
| 세금계산서 발행 상태 | 회계 담당자에게는 정산 완료의 핵심 후속 업무다 | 정산 번들에서 계산서 발행 상태로 이어지는 플로우를 시각화 |
| 화물맨 연동 | 외부 플랫폼 연동은 반복 등록을 줄이는 명확한 가치다 | “한 번 입력 후 화물맨 전송/상태 확인”으로 설명 |

## 5. 현재 랜딩 진단

| 영역 | 유지할 점 | 개선할 점 |
| --- | --- | --- |
| hero | AI 오더 등록 애니메이션이 서비스의 자동화 가치를 잘 보여준다 | hero 자체를 바꾸기보다 다음 섹션이 hero의 약속을 이어받아야 한다 |
| 기능 섹션 | 주문, 정산, 세금계산서, AI, 지도, 연동 축이 이미 있다 | 기능 카드만으로는 “업무가 어떻게 흘러가는지”가 약하다 |
| 제품 섹션 | `OPTIC Broker`, `OPTIC Shipper` 등 역할별 제품 구분이 있다 | Broker/Shipper를 실제 업무 시나리오와 연결해야 한다 |
| 연동 섹션 | AI 등록, 전자세금계산서, 화물맨 등 핵심 연동 흐름이 있다 | 외부 연동이 어떤 수작업을 줄이는지 더 구체화해야 한다. `화물맨`은 고유명으로 유지하고, 그 외 외부 서비스명은 일반 기능명으로 처리한다 |
| 헤더 CTA | `도입 문의하기`는 전환 CTA로 유지 가능하다 | 실제 서비스로 이동하는 CTA를 추가하되 `서비스 테스트` 표현은 피해야 한다 |
| 문구 상태 | 현재 코드 기준 일부 문구가 짧고 설명 중심성이 부족하다 | 실제 화면에서 한글 표시와 문구 톤을 재확인해야 한다 |

## 6. 랜딩 메시지의 중심축

추천 메시지 축은 다음과 같다.

> AI로 운송 오더를 빠르게 등록하고, 상하차지와 외부 연동, 정산 번들, 세금계산서까지 한 흐름으로 이어지는 운송 운영 시스템

이 메시지는 hero의 AI 등록 장면을 유지하면서도, 페이지 아래로 내려갈수록 “그 다음 업무가 어떻게 자동화되는지”를 설명할 수 있다.

브랜드 문장으로 압축하면 “오더부터 정산까지 한눈에.”가 가장 적합하다. 영문 보조 문구가 필요하면 `From order to settlement, clearly.`를 사용할 수 있다.
