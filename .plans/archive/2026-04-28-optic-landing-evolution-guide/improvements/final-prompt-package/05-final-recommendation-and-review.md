# 05. 최종 추천안과 리뷰

## 1. 최종 추천안

최종 추천은 **업무 매뉴얼형 스크롤 랜딩 + 핵심 자동화 애니메이션 + 실제 서비스 이동 CTA**다.

이 방향은 현재 hero의 AI 오더 등록 강점을 유지하면서, 페이지 아래에서 상하차지 관리, 화물맨 연동, 정산 번들화, 세금계산서 발행을 차례로 설명할 수 있다.

## 2. 왜 이 안을 추천하는가

| 기준 | 판단 |
| --- | --- |
| 현재 hero와의 연결 | AI가 등록한 Order가 아래 섹션의 운영 흐름으로 이어져 자연스럽다 |
| 서비스 이해도 | 기능 카드보다 실제 업무 순서를 보여주므로 처음 보는 사람도 이해하기 쉽다 |
| 구현 비용 | 전체 3D/고난도 애니메이션보다 단계별 카드/타임라인 애니메이션으로 MVP가 가능하다 |
| 영업 가치 | 문의 CTA와 실제 서비스 이동 CTA를 분리해 방문자 의도별 전환을 받을 수 있다 |
| 문서 근거 | 주선사/화주 매뉴얼과 도메인 가이드의 실제 흐름을 기반으로 구성 가능하다 |
| 브랜드 안정성 | 고객 화면은 `OPTIC`, 내부 엔진 설명은 `OPTICS`로 분리해 이중 브랜드 혼란을 줄인다 |

## 3. MVP 범위

| 우선순위 | 항목 | 포함 내용 |
| ---: | --- | --- |
| 1 | 헤더 로고 적용 | 제공 이미지 기반 `OPTIC` 워드마크를 헤더 로고 기준으로 반영 |
| 2 | 헤더 CTA 개선 | `OPTIC 바로가기` 외부 링크 추가, 기존 `도입 문의하기` 유지 |
| 3 | hero 이후 overview | AI 오더 등록 이후 업무가 이어지는 5단계 흐름 설명 |
| 4 | 상하차지 관리 섹션 | 주소록, 담당자, 연락처, 메모 재사용 가치 설명 |
| 5 | 외부 연동 섹션 | 화물맨 연동을 “중복 입력 감소”로 설명 |
| 6 | 매출 정산 번들 섹션 | 완료 주문이 SalesBundle로 묶이는 장면 구현 |
| 7 | 세금계산서 섹션 | 정산 묶음에서 계산서 발행 상태로 이어지는 장면 구현 |
| 8 | 카피 정리 | `OPTIC` 표기 통일, `Cargo` 중심 언어를 줄이고 오더/운송/조율 가치 중심으로 문구 수정 |
| 9 | footer/About 정리 | 필요한 경우 `Powered by OPTICS`만 제한적으로 노출 |

## 4. 확장 범위

| 항목 | 확장 아이디어 |
| --- | --- |
| 매입 정산 | 기사/운송사 기준 PurchaseBundle 흐름까지 별도 섹션화 |
| 역할별 보기 | Broker/Shipper 탭으로 같은 운송 흐름을 역할별로 전환 |
| 인터랙티브 매뉴얼 | 스크롤이 아니라 사용자가 직접 단계 버튼을 눌러 업무 흐름 확인 |
| 실제 서비스 deep link | 추후 로그인/권한 구조가 안정되면 특정 화면으로 직접 연결 |
| 성과 지표 | 주문 등록 시간, 정산 누락 감소, 세금계산서 처리 시간 같은 지표 추가 |
| 브랜드 도메인 정리 | `{service}.{tenant}.{root}` 기준으로 `brkr`, `shpr`, `ops`, `fin` 도메인 후보 확정 |
| 로고 시스템 확장 | symbol-only, favicon, Open Graph, dark mode용 로고 자산 확장 |

## 5. 구현 전 확인 질문

| 질문 | 필요한 이유 |
| --- | --- |
| 제공된 로고 이미지를 production 기준 시안으로 확정할 것인가? | AI 생성 이미지 기반이므로 상표/권리, 원본성, 최종 디자인 승인 여부를 확인해야 한다 |
| 로고는 SVG로 재제작할 것인가, PNG 파생 자산으로 시작할 것인가? | 헤더와 favicon에서 선명도를 유지하려면 벡터화 여부를 결정해야 한다 |
| `https://mm-broker-test.vercel.app/`를 공개 랜딩에서 바로 연결해도 되는가? | URL은 실제 서비스 이동 CTA가 되므로 접근 권한과 공개 가능성을 확인해야 한다 |
| `OPTIC 바로가기` 클릭 시 새 탭이 맞는가? | 랜딩 이탈을 막으려면 새 탭이 기본값으로 적합하다 |
| 화물맨 연동은 전송 상태, 오류 로그, 재전송 중 어디까지 보여줄 것인가? | 애니메이션 범위와 구현 난이도에 영향을 준다 |
| 매출 정산만 먼저 보여줄 것인가, 매입 정산도 함께 보여줄 것인가? | MVP 범위와 애니메이션 복잡도가 달라진다 |
| 현재 hero 문구는 그대로 유지할 것인가, `OPTIC` 표기와 CTA만 정리할 것인가? | hero는 유지하되 문구 보강 가능성이 있다 |
| footer의 `Powered by OPTICS` 노출 위치를 유지할 것인가? | `OPTICS`는 고객 화면 전면이 아니라 보조 위치에서만 안전하다 |
| 추후 공식 도메인은 `{service}.{tenant}.{root}` 규칙으로 이동할 것인가? | 현재 Vercel URL과 브랜드 도메인 정책의 간극을 줄여야 한다 |

## 6. 적용 체크리스트

| 항목 | 상태 | 메모 |
| --- | --- | --- |
| 브랜드명 `OPTIC` 기준 적용 | 완료 | 문서 패키지 전체에 반영 |
| `OPTIC/OPTICS` 역할 분리 | 완료 | 고객 제품/내부 엔진 구분 반영 |
| 제공 로고 이미지 반영 | 완료 | 별도 로고 자산 적용 가이드 추가 |
| `Cargo` 중심 언어 축소 | 완료 | `오더`, `운송`, `조율`, `운영 흐름` 중심으로 보강 |
| 화물맨 표기 통일 | 완료 | 외부 연동 문구를 `화물맨` 기준으로 정리 |
| 외부 브랜드명 노출 제한 | 완료 | `화물맨`을 제외한 외부 서비스명은 일반 기능명으로 정리 |
| `서비스 테스트` 문구 제외 | 완료 | 실제 서비스 이동 CTA로 재정의 |
| 실제 서비스 URL 반영 | 완료 | `https://mm-broker-test.vercel.app/` |
| hero 유지 조건 반영 | 완료 | hero 자체 변경은 MVP에서 제외 |
| 상하차지/연동/정산/계산서 반영 | 완료 | 각 기능별 섹션과 애니메이션 제안 포함 |
| 참고 문서 근거 연결 | 완료 | 도메인 가이드, broker/shipper manual, feature index 기반 |
| 코드 수정 금지 | 완료 | 이번 산출물은 문서만 작성 |

## 7. Self-review

| 항목 | Severity | Confidence | Action | 메모 |
| --- | --- | --- | --- | --- |
| AI 생성 로고의 권리/상표 검토 필요 | high | likely | needs-user-input | production 로고로 확정하기 전 사용 권리, 상표 충돌, 최종 디자인 승인 여부를 확인해야 한다 |
| 로고 원본이 raster 이미지라 선명도 저하 가능 | medium | likely | queued | 구현 전 SVG 또는 고해상도 투명 PNG로 파생 자산을 만들어야 한다 |
| 실제 서비스 URL 공개 가능성 미확인 | medium | likely | needs-user-input | 공개 랜딩에서 바로 연결하기 전 접근 권한과 공개 범위를 확인해야 한다 |
| 화물맨 표기 통일 | low | confirmed | auto-fixed | 외부 연동 문구를 `화물맨` 기준으로 정리했다 |
| 외부 브랜드명 노출 제한 | low | confirmed | auto-fixed | `화물맨`을 제외한 외부 서비스명은 일반 기능명으로 정리했다 |
| 현재 랜딩 소스의 한글 표시 상태 확인 필요 | medium | tentative | needs-verification | 도구 출력에서 일부 문구가 깨져 보였으므로 실제 브라우저 표시를 확인해야 한다 |
| 매입 정산 포함 범위 | low | likely | queued | MVP는 매출 정산 중심, 확장으로 매입 정산을 제안 |
| 공식 도메인 정책과 현재 URL 차이 | medium | likely | needs-user-input | 브랜드 가이드는 `{service}.{tenant}.{root}`를 권장하지만 현재 CTA URL은 Vercel 테스트/실서비스 URL이다 |

종료 조건: 문서 반영은 완료됐지만, production 로고 확정 전에는 권리/상표 검토가 필요하다.
로고 사용 승인 이슈는 구현 전 체크로 분리하고, 나머지 항목은 공개 URL 확인, 용어 통일, 브라우저 표시 확인으로 넘긴다.
