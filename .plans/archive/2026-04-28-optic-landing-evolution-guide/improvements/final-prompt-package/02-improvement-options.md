# 02. 랜딩 개선 콘셉트 비교안

## 1. 비교 요약

| 안 | 콘셉트 | 추천도 | 구현 난이도 | 가장 적합한 상황 |
| --- | --- | --- | --- | --- |
| A | 업무 매뉴얼형 스크롤 랜딩 | high | medium | 현재 hero를 유지하면서 서비스 이해도를 가장 크게 높이고 싶을 때 |
| B | 역할별 진입 랜딩 | medium | medium | 주선사/화주 각각의 서비스 진입을 강하게 나누고 싶을 때 |
| C | 자동화 파이프라인 랜딩 | high | high | 애니메이션 중심으로 “입력부터 정산까지”를 강하게 보여주고 싶을 때 |
| D | 정산 신뢰 중심 랜딩 | medium | medium | 정산/세금계산서가 구매 결정의 핵심일 때 |
| E | 실제 서비스 체험 유도 랜딩 | medium | low | 실제 서비스 링크 클릭을 빠르게 늘리고 싶을 때 |

## 2. A안: 업무 매뉴얼형 스크롤 랜딩

| 항목 | 내용 |
| --- | --- |
| 핵심 아이디어 | hero의 AI 오더 등록 이후, 실제 운영 매뉴얼처럼 업무가 이어지는 구조 |
| 적합한 방문자 | 주선사 운영자, 화주 운영자, 도입 검토자 |
| 섹션 흐름 | AI 오더 등록 → 상하차지 관리 → 배차/외부 연동 → 정산 번들 → 세금계산서 → 실제 서비스 이동 |
| 애니메이션 아이디어 | 스크롤 위치에 따라 카드, 지도, 상태 배지, 정산 묶음, 계산서가 순서대로 완성 |
| 문구 방향 | 설명형 + 업무 흐름형 |
| 장점 | 서비스가 무엇을 해주는지 가장 쉽게 이해된다 |
| 단점/리스크 | 섹션이 길어질 수 있어 밀도 조절이 필요하다 |
| 구현 난이도 | medium |
| 추천도 | high |

추천 이유: 사용자가 요청한 “매뉴얼처럼 보여주는” 방향과 가장 잘 맞는다.

## 3. B안: 역할별 진입 랜딩

| 항목 | 내용 |
| --- | --- |
| 핵심 아이디어 | 방문자가 `주선사` 또는 `화주` 중 자기 역할을 먼저 선택하고 해당 흐름을 본다 |
| 적합한 방문자 | 역할이 분명한 실무자 |
| 섹션 흐름 | hero → 주선사/화주 선택 → 역할별 업무 흐름 → 공통 정산/현황 → CTA |
| 애니메이션 아이디어 | 좌우 탭 또는 segmented control로 Broker/Shipper 플로우 전환 |
| 문구 방향 | 사용자 역할 중심 |
| 장점 | 방문자가 자기 업무와 직접 연결하기 쉽다 |
| 단점/리스크 | `OPTIC` 제품 브랜드와 역할별 제품명이 분산되어 보일 수 있다 |
| 구현 난이도 | medium |
| 추천도 | medium |

이 안은 서비스가 두 사용자군을 모두 강하게 공략해야 할 때 좋다. 다만 현재 hero가 주선사 AI 오더 등록에 가까우므로, 화주 흐름과 연결하는 보조 설명이 필요하다.

## 4. C안: 자동화 파이프라인 랜딩

| 항목 | 내용 |
| --- | --- |
| 핵심 아이디어 | 하나의 운송 건이 등록, 연동, 진행, 정산, 세금계산서로 변환되는 과정을 파이프라인처럼 보여준다 |
| 적합한 방문자 | 운영 효율과 자동화에 관심이 큰 의사결정자 |
| 섹션 흐름 | hero → 파이프라인 overview → 각 단계 deep dive → ROI/신뢰 → CTA |
| 애니메이션 아이디어 | 운송 카드가 상태 배지를 바꾸며 다음 단계로 이동하고, 마지막에 정산 번들과 계산서로 변환 |
| 문구 방향 | 전환형 + 자동화 가치 중심. 고객 문구는 `OPTIC`, 내부 근거는 `OPTICS` 엔진으로 분리 |
| 장점 | “수작업이 시스템 흐름으로 바뀐다”는 메시지가 강하다 |
| 단점/리스크 | 애니메이션 구현 난이도와 QA 비용이 높다 |
| 구현 난이도 | high |
| 추천도 | high |

이 안은 가장 인상적이지만 구현 비용이 있다. MVP에서는 A안을 먼저 만들고, C안의 핵심 애니메이션만 가져오는 조합이 좋다.

## 5. D안: 정산 신뢰 중심 랜딩

| 항목 | 내용 |
| --- | --- |
| 핵심 아이디어 | 주문보다 정산, 청구, 지급, 세금계산서 안정성을 중심으로 설득한다 |
| 적합한 방문자 | 대표, 정산 담당자, 회계 담당자 |
| 섹션 흐름 | hero → 정산 문제 → 매출/매입 구조 → 번들 정산 → 세금계산서 → 리스크 감소 |
| 애니메이션 아이디어 | 흩어진 주문들이 SalesBundle/PurchaseBundle로 묶이고 상태가 입금/송금/완료로 이동 |
| 문구 방향 | 신뢰형 + 리스크 감소형 |
| 장점 | 돈과 증빙에 직접 닿는 고통을 잘 보여준다 |
| 단점/리스크 | AI 등록 hero와 연결이 약해질 수 있다 |
| 구현 난이도 | medium |
| 추천도 | medium |

정산이 도입 결정의 핵심이면 강력하다. 다만 첫 화면의 AI 메시지를 살리려면 후반부 섹션으로 쓰는 편이 좋다.

## 6. E안: 실제 서비스 체험 유도 랜딩

| 항목 | 내용 |
| --- | --- |
| 핵심 아이디어 | 헤더와 CTA에서 실제 서비스 진입을 강하게 노출한다 |
| 적합한 방문자 | 이미 소개를 받았거나 빠르게 서비스를 확인하려는 사용자 |
| 섹션 흐름 | hero → 실제 서비스 바로가기 → 핵심 기능 요약 → 문의 CTA |
| 애니메이션 아이디어 | 최소화. 버튼, hover, preview card 중심 |
| 문구 방향 | 행동 유도형 |
| 장점 | 구현이 빠르고 실제 서비스 사용으로 바로 이어질 수 있다 |
| 단점/리스크 | 서비스 이해도가 낮은 방문자는 바로 이탈할 수 있다 |
| 구현 난이도 | low |
| 추천도 | medium |

이 안은 단독 전략보다 A안 또는 C안에 CTA 요소로 섞는 것이 좋다.

## 7. 최종 비교 결론

추천 조합은 **A안 + C안 일부 + E안 CTA**다.

즉, 기본 구조는 업무 매뉴얼형 스크롤 랜딩으로 만들고, 핵심 장면에는 자동화 파이프라인 애니메이션을 넣는다. 고객이 보는 전면 브랜드는 `OPTIC`으로 유지하고, footer/About나 내부 구현 가이드에서만 `Powered by OPTICS` 또는 `OPTICS Ledger` 같은 엔진 표현을 보조적으로 사용한다. 헤더에는 `OPTIC 바로가기` 같은 실제 서비스 이동 CTA를 추가해, 설명을 충분히 읽은 사람과 바로 들어가고 싶은 사람 모두를 받을 수 있게 한다.
