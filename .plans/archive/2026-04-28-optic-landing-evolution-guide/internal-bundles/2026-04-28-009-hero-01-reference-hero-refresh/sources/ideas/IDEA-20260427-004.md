## 1. 아이디어 요약

현재 `hero-liquid-gradient-background` 문서대로 구현된 Hero 배경은 사용자가 원하는 시각 결과와 다르다. 새 개선은 `.references/design/hero-01` 레퍼런스를 기준으로 Hero 섹션을 다시 설계한다.

핵심은 `hero-01`의 몰입형 첫 화면, 중앙 집중형 타이포그래피, full-bleed liquid gradient, pointer 반응형 움직임, 높은 시각적 밀도를 현재 landing의 light mode / dark mode theme에 맞게 재해석하는 것이다.

단, 이번 단계는 구현이 아니라 planning pipeline 등록이다. 실제 코드는 draft, bridge, dev 단계에서 다룬다.

## 2. 배경

이전 아이디어 `IDEA-20260427-001`은 CodePen 스타일의 interactive liquid gradient를 Hero 배경에 도입하는 것이 목표였다. 최종 구현은 CSS-first MVP로 마무리됐고, `Three.js` / WebGL 구현은 deferred로 남았다.

## 4. hero-01 레퍼런스 분석

| 요소 | 관찰 내용 | 현재 landing에 가져올 의미 |
|---|---|---|
| 화면 구조 | `canvas`가 viewport 전체를 덮고, Hero 텍스트가 중앙에 고정된다. | Hero 첫 화면이 배경 장식이 아니라 주 시각 경험처럼 보여야 한다. |
| 타이포그래피 | `Liquid Gradient` 대형 heading이 화면 중심을 지배한다. | 현재 headline도 더 과감한 scale, line-height, 중심 정렬을 검토한다. |
| 배경 질감 | 여러 blob이 canvas에서 계속 움직이고 pointer 위치에 반응한다. | CSS-only blur blob보다 더 유동적인 depth와 반응성을 목표로 한다. |
| 색상 시스템 | 5개 scheme과 color adjuster가 있고, 강한 보색 조합을 사용한다. | scheme UI는 제외하되, light/dark theme별 palette 후보를 고정한다. |
| Interaction | pointer movement가 색 field와 custom cursor에 반영된다. | custom cursor는 제외하되 pointer-reactive gradient 여부는 검토한다. |
| Controls | color scheme buttons, Colors panel, copy/export 기능이 있다. | 랜딩 Hero에서는 제품 UI로 오해될 수 있으므로 기본 제외한다. |
| Responsive | 모바일에서는 heading 크기와 control 위치를 줄인다. | 현재 DashboardPreview가 있으므로 mobile Hero height와 overflow를 별도 검증한다. |

## 5. 현재 구현과의 gap

| Gap | 현재 상태 | 원하는 방향 |
|---|---|---|
| 시각 우선순위 | gradient가 Hero 뒤의 보조 장식으로 보인다. | `hero-01`처럼 첫 화면의 주요 인상으로 느껴져야 한다. |
| 움직임의 질감 | CSS radial gradient 3개가 scale/translate/rotate로 움직인다. | blob이 더 많고, 색이 섞이며, pointer에 반응하는 liquid field가 필요하다. |
| 레퍼런스 충실도 | `hero-01`의 full-viewport canvas 구조와 다르다. | full-bleed background 또는 canvas-like layer로 구조를 다시 잡는다. |
| 색 조합 | `--landing-accent-start/end` 중심의 보라/파랑 계열이다. | light/dark theme별로 base, accent, highlight가 분리된 palette가 필요하다. |
| Hero 구성 | headline, subtitle, CTA, DashboardPreview가 모두 같은 섹션에 있다. | 레퍼런스 copy 느낌과 제품 landing conversion 요소의 균형을 다시 정의한다. |
| UI controls | 레퍼런스에는 scheme controls가 있지만 현재 Hero에는 없다. | controls는 기본 제외하고, 개발/QA용 hidden story 또는 debug 후보로만 둔다. |

## 6. 개선 방향

추천 방향은 `hero-01` visual copy fidelity를 우선하고, 현재 landing의 제품 목적을 유지하는 것이다.

1. Hero background는 `hero-01`처럼 viewport를 채우는 주 시각 레이어로 재설계한다.
2. 현재 CTA와 `DashboardPreview`는 유지하되, 첫 viewport에서 gradient와 headline의 인상이 묻히지 않도록 배치 우선순위를 조정한다.
3. 색상은 reference palette를 그대로 쓰지 않고 `:root`, `[data-theme="dark"]` token에서 파생한다.
4. 구현 방식은 CSS-only 유지, Canvas 2D, WebGL/Three.js 중 하나를 draft에서 비교한다.
5. pointer-reactive 효과는 desktop에서만 우선 검토하고, mobile과 `prefers-reduced-motion`에서는 static 또는 저강도 fallback을 둔다.

## 7. Light/Dark 색상 방향

아래 palette는 구현 확정값이 아니라 draft에서 검증할 후보이다. 핵심은 기존 theme의 흰색/검정 배경과 보라/파랑 accent를 유지하면서, `hero-01`처럼 색 field가 더 풍부하게 보이도록 보조색을 추가하는 것이다.

| Theme | Base | Primary field | Secondary field | Highlight | Guardrail |
|---|---|---|---|---|---|
| light | `--landing-background` `#ffffff` | violet 계열 `--landing-accent-start` | blue 계열 `--landing-accent-end` | cyan 또는 soft magenta | CTA와 headline 대비가 WCAG AA를 유지해야 한다. |
| dark | `--landing-background` `#0a0a0a` | vivid violet | electric blue | mint/cyan 또는 warm coral | 화면 전체가 보라/파랑 단색으로 뭉치지 않아야 한다. |

### 후보 token 값

| Token | Light 후보 | Dark 후보 | 의도 |
|---|---|---|---|
| `--hero-field-base` | `#ffffff` | `#0a0a0a` | 기존 theme 배경 유지 |
| `--hero-field-primary` | `#7c3aed` at 30-36% | `#9333ea` at 48-56% | 기존 violet accent 계승 |
| `--hero-field-secondary` | `#2563eb` at 24-30% | `#3b82f6` at 38-46% | 기존 blue accent 계승 |
| `--hero-field-highlight` | `#06b6d4` at 14-20% | `#22d3ee` at 20-28% | cyan으로 차가운 빛과 깊이 추가 |
| `--hero-field-warm` | `#f472b6` at 8-14% | `#fb7185` at 10-16% | violet/blue 단색화를 피하는 보조 warm accent |
| `--hero-field-contrast-veil` | `rgba(255, 255, 255, 0.52)` | `rgba(0, 0, 0, 0.34)` | 텍스트 대비와 CTA 시인성 보호 |

light mode는 깨끗한 white surface 위에 낮은 opacity의 color field를 얹고, dark mode는 더 높은 saturation을 허용한다. 다만 두 모드 모두 보라/파랑만 반복되는 one-note palette가 되지 않도록 cyan 또는 warm accent를 최소 1개 이상 포함한다.

권장 token 후보:

- `--hero-field-base`
- `--hero-field-primary`
- `--hero-field-secondary`
- `--hero-field-highlight`
- `--hero-field-contrast-veil`
- `--hero-field-opacity`
- `--hero-field-blur`
