## 1. 문제 요약

Phase 3 preview의 order form은 세로 밀도와 hit-area 정확도에서 후속 정비가 필요하다. 특히 pickup/delivery `DateTimeCard` 배치, 19개 hit-area 좌표, Tablet 좌표 분리 여부, overlay anchor 위치가 실제 DOM과 어긋날 수 있다.

## 2. 목표

- Col 2 내부 pickup/delivery `DateTimeCard` 를 2열 구조로 정리한다.
- `src/components/dashboard-preview/hit-areas.ts` 의 19개 bounds를 실제 DOM 기준으로 재측정한다.
- Tablet bounds를 desktop과 계속 공유할지, 별도로 분리할지 결정한다.
- `interactive-overlay.tsx` anchor 위치를 ScaledContent 내부 기준으로 재검토한다.

## 3. 예상 범위

| 영역 | 예상 작업 |
|---|---|
| Layout | `order-form/index.tsx`, `datetime-card.tsx` 구조 조정 |
| Hit area | `hit-areas.ts` bounds 재측정 및 테스트 갱신 |
| Overlay | `interactive-overlay.tsx` anchor 기준 검토 |
| Test | hit-area, dashboard-preview, layout 관련 회귀 테스트 갱신 |
