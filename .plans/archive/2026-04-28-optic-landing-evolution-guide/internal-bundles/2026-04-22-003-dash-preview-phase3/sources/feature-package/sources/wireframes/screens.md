## 0. 이 wireframe이 다루는 것

**단일 뷰 내부의 Step 상태 전환**이 본 wireframe의 중심 주제다. 일반적인 multi-page navigation wireframe과 달리, 하나의 Dashboard Preview 영역 안에서 4개 Step이 6~8초 주기로 순환 재생되며, 각 Step마다 UI 상태 + 조작감 레이어 발생 지점이 다르다.

따라서 본 문서는 **"Step 4종 × Viewport 3종"** 매트릭스로 wireframe을 구성한다. "화면 목록"은 Step 상태 스냅샷 목록으로 대체된다.

### 3-2. Tablet Step별 차이

- **STATE-001 ~ STATE-004b**: Desktop과 **동일한 DOM 구조(3-column) + 동일 조작감 타이밍**. scale factor만 0.45 → **0.40**으로 변경 (기존 0.32에서 상향 조정, 가독성 우선).
- **3-column 유지 의도**: 원본은 `lg:grid-cols-3` breakpoint(≥1024px)에서만 3-col이고 Tablet(<1024px)에서 1-col reflow하지만, Phase 3는 "랜딩 대표 화면은 **데스크탑 구조를 압축 표시**"가 마케팅 가치가 더 크다는 판단으로 C안 채택. 원본과의 divergence는 **의도적**이며 REQ-DASH-023/024에 명시.
- **조작감 대각 흐름 유지**: Col 1 → Col 2 → Col 3 흐름이 Tablet에서도 시각적으로 성립 (각 Column 너비 ~100~120 CSS px로 축소되나 border pulse로 시선 유도).
- **텍스트 가독성 (R5 리스크)**: 필드 레이블이 매우 작아짐 (scale 0.40 + 3-col = 원본 각 Col 대비 1/3 너비). Milestone 5 A/B 검증 대상. 실패 시 Tablet scale 0.45 상향 또는 원본과 동일 1-column reflow 폴백 옵션 보유.
- **UI 힌트 사이즈**: text-xs (12px) → text-[10px]. 클램프 하한 유지.
- **히트 영역 크기**: 터치 최소 44×44 CSS px 보장 — scale 0.40 환경에서 원본 기준 110×110 px 상당 영역 확보 (기존 0.32 대비 여유도 개선).
