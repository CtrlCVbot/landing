## 1. 문제 요약

F3에서 업무 매뉴얼형 섹션의 6단계 구조를 만들었지만, 아직 각 단계가 실제 운영 화면처럼 움직이거나 상태가 변하는 느낌은 약하다. 방문자가 `AI 오더 등록 → 상하차지 관리 → 배차/운송 상태 → 화물맨 연동 → 정산 자동화 → 세금계산서 관리`를 더 빠르게 이해하려면 단계별 등장 애니메이션과 샘플 상태 표현이 필요하다.

## 2. 목표

- F3의 stable workflow id와 순서를 유지한다.
- 단계 카드가 과하지 않게 순차 등장하고, hover/focus에서 레이아웃이 흔들리지 않게 한다.
- 화물맨 전송 성공/오류, SalesBundle 묶음, 세금계산서 상태를 mock 상태로 보여준다.
- 실제 API, 실시간 연동, 자동 발행 완료를 약속하지 않는다.
- `prefers-reduced-motion` 사용자를 위해 움직임을 줄일 수 있게 한다.

## 3. 예상 범위

| 영역 | 예상 작업 |
|---|---|
| Motion | `src/lib/motion.ts`에 workflow 전용 variant 추가 |
| Data | `src/lib/landing-workflow.ts`에 mock 상태 데이터 추가 |
| UI | `src/components/sections/workflow-manual.tsx` 상태 보드와 motion 적용 |
| Tests | workflow data/component tests 보강 |
| QA | typecheck, test, lint, build, desktop/mobile preview 확인 |

## 4. Acceptance Criteria

- workflow step 순서와 title은 F3와 동일하다.
- `샘플 상태 보드`가 렌더링된다.
- `화물맨 전송 성공`, `필드 오류 재확인`, `SalesBundle 묶음 생성`, `세금계산서 상태 확인`이 보인다.
- `실시간 자동 연동`, `자동 발행 완료`, `성공 보장` 같은 과장 문구가 없다.
- reduced motion 환경에서도 콘텐츠가 모두 보인다.
